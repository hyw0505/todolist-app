## 발견된 문제

### 🔴 문제 1: Access Token 만료 및 Refresh Token 오류

**증상:**
- 로그인 후 할일 목록 조회 시 401 Unauthorized 에러 발생
- `/api/v1/auth/refresh` 요청도 401 반환
- 할일 목록이 "로딩 중" 상태에서 멈춤

**에러 로그:**
```
[ERROR] Failed to load resource: the server responded with a status of 401 (Unauthorized)
  @ http://localhost:3000/api/v1/todos?sort_by=due_date&sort_order=asc&page=1&limit=10

[ERROR] Failed to load resource: the server responded with a status of 401 (Unauthorized)
  @ http://localhost:3000/api/v1/auth/refresh
```

**원인 분석:**
1. 로그인 시 발급된 Access Token 이 빠르게 만료됨 (15 분)
2. Refresh Token 처리 로직에 문제 발생
3. 백엔드 JWT secret 키 설정 확인 필요

**백엔드 설정 확인 필요:**
- `JWT_ACCESS_SECRET` 환경 변수
- `JWT_REFRESH_SECRET` 환경 변수
- 토큰 발급/검증 로직

## UI/UX 확인 사항

### ✅ 긍정적 관찰

1. **헤더 일관성:** 모든 페이지에서 헤더가 일관되게 표시됨
   - 로고 (todolist-app)
   - 테마 토글 버튼 (☀️/🌙)
   - 사용자명 + 로그아웃 버튼

2. **반응형 폼:** 회원가입 폼이 실시간 유효성 검사 제공
   - 비밀번호 정책 안내 (8-64 자, 대소문자, 숫자, 특수문자)
   - 입력 필드 placeholder 명확함

3. **모달 UI:** 할일 추가 모달이 명확한 레이블과 입력 필드 제공

### ⚠️ 개선 필요 사항

1. **로딩 상태 가시성:** 
   - "할일을 불러오는 중..." 메시지만 표시
   - 로딩 스피너 시각적 피드백 강화 필요

2. **에러 핸들링:**
   - 401 에러 발생 시 사용자 친화적 메시지 없음
