## UI/UX 확인 사항

### ✅ 긍정적 관찰

1. **헤더 일관성:** 모든 페이지에서 헤더가 일관되게 표시됨
   - 로고 (todolist-app)
   - 테마 토글 버튼 (☀️/🌙)
   - 사용자명 + 로그아웃 버튼

2. **반응형 폼:** 회원가입 폼이 실시간 유효성 검사 제공
   - 비밀번호 정책 안내 (8-64 자, 대소문자, 숫자, 특수문자)
   - 입력 필드 placeholder 명확함
   - 비밀번호 확인 일치 시에만 가입 버튼 활성화

3. **모달 UI:** 할일 추가 모달이 명확한 레이블과 입력 필드 제공
   - 제목 (필수), 설명 (선택), 시작일 (필수), 종료일 (필수)
   - 취소/추가 버튼 명확

4. **상태 자동 산출:** 할일 생성 시 시작일/종료일 기반 상태 자동 계산
   - 시작일 ≤ 오늘 ≤ 종료일 → "진행 중"
   - 오늘 < 시작일 → "시작 전"
   - 오늘 > 종료일 → "기한 초과" (테스트 필요)

5. **할일 목록 표시:** 
   - 할일 카드에 제목, 상태, 설명, 기간 표시
   - 완료/수정/삭제 버튼 제공
   - 페이지네이션 UI 표시

### ⚠️ 개선 필요 사항

1. **로딩 상태 가시성:** 
   - API 요청 중 로딩 스피너 시각적 피드백 강화 필요
   - 현재는 텍스트만 표시 ("할일을 불러오는 중...")

2. **에러 핸들링:**
