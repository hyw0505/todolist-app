### 1.3 디자인 원칙

| 원칙 | 설명 |
|------|------|
| **단순성** | 불필요한 요소를 제거하고 핵심 기능에 집중 |
| **명확성** | 상태와 액션이 직관적으로 이해 가능 |

## 5. 상태 관리 개요 (Zustand)

### 5.1 인증 스토어 (`useAuthStore`)

```typescript
interface AuthState {
  accessToken: string | null;
  user: User | null;
  isAuthenticated: boolean;
  
  // Actions
  setAuth: (token: string, user: User) => void;
  clearAuth: () => void;
}
```

**상태 흐름:**
```
[초기] accessToken: null, user: null, isAuthenticated: false
   │
   │ 로그인 성공
   ▼
[인증됨] accessToken: "JWT", user: {...}, isAuthenticated: true
   │
   │ 로그아웃 / 토큰 만료
   ▼
[초기] accessToken: null, user: null, isAuthenticated: false
```

### 5.2 할일 필터 스토어 (`useTodoFilterStore`)

```typescript
interface TodoFilterState {
  status: TodoStatus | 'ALL';
  sortBy: 'start_date' | 'due_date';
  sortOrder: 'asc' | 'desc';
  page: number;
  limit: number;

  // Actions
  setStatus: (status: TodoStatus | 'ALL') => void;
  setSortBy: (sortBy: 'start_date' | 'due_date') => void;
  setSortOrder: (order: 'asc' | 'desc') => void;
  setPage: (page: number) => void;
  reset: () => void;
}
```

**초기값:**
- `status`: 'ALL'
- `sortBy`: 'due_date'
- `sortOrder`: 'asc'
- `page`: 1
- `limit`: 10

### 5.3 테마 스토어 (`useThemeStore`)

```typescript
interface ThemeState {
  theme: 'light' | 'dark' | 'system';

  // Actions
  setTheme: (theme: 'light' | 'dark' | 'system') => void;
  toggleTheme: () => void; // light ↔ dark 토글
}
```

**초기값:**
- `theme`: 'system' (시스템 설정 따름)

**상태 흐름:**
```
[초기] theme: 'system'
   │
   │ 사용자가 다크모드 토글
   ▼
[다크모드] theme: 'dark'
   │
   │ 사용자가 라이트모드 토글
   ▼
[라이트모드] theme: 'light'
   │
   │ 사용자가 시스템 설정 선택
   ▼
[시스템] theme: 'system' → 시스템 테마 감지
```

**localStorage 저장:**
- 키: `todolist-theme`
- 값: `'light' | 'dark' | 'system'`
- 목적: 사용자 테마 선호도 영구 저장

**시스템 테마 감지:**
```typescript
// prefers-color-scheme 미디어 쿼리 감지
const mediaQuery = window.matchMedia('(prefers-color-scheme: dark)');
const systemTheme = mediaQuery.matches ? 'dark' : 'light';
```

### 5.4 언어 스토어 (`useLanguageStore`)

```typescript
type Language = 'ko' | 'en' | 'jp';

interface LanguageState {
  language: Language;

  // Actions
  setLanguage: (lang: Language) => void;
}
```

**초기값:**
- `language`: 사용자 프로필 `language` 필드 → 브라우저 `Accept-Language` → `'ko'` 순으로 결정

**상태 흐름:**
```
[초기] language: 사용자 프로필 or 브라우저 감지 or 'ko'
   │
   │ 사용자가 언어 선택 (LanguageSelector)
   ▼
[변경] language: 'en' | 'jp' | 'ko'
       → i18next.changeLanguage() 호출
       → localStorage 저장 ('i18nextLng' 키)
```

**localStorage 저장:**
- 키: `i18nextLng`
- 값: `'ko' | 'en' | 'jp'`
- 목적: 새로고침 시 언어 설정 유지

#### 6.1.3 입력 필드 검증 규칙

**이메일:**
- 필수 입력
- 유효한 이메일 형식 (`/^[^\s@]+@[^\s@]+\.[^\s@]+$/`)
- 오류 메시지: "유효한 이메일 주소를 입력해주세요"

**비밀번호:**
- 필수 입력
- 8 자 이상 64 자 이하
- 오류 메시지: "비밀번호를 입력해주세요"

#### 6.1.4 버튼 상태

| 상태 | 조건 | 비주얼 |
|------|------|--------|
| default | 초기 상태 | 파란색 배경, 흰색 텍스트 |
| hover | 마우스 오버 | 약간 어두운 파란색 |
| active | 클릭 중 | 더 어두운 파란색 |
| disabled | 입력값 없음 | 회색 배경, 커서 not-allowed |
| loading | 제출 중 | 스피너 아이콘, "로그인 중..." 텍스트 |

- 오류 메시지:
  - 형식 오류: "유효한 날짜를 입력해주세요"
  - 과거 날짜: "오늘 이후의 날짜를 선택해주세요"

- 필수 입력

- 오류 메시지:
  - 형식 오류: "유효한 날짜를 입력해주세요"

## 9. 반응형 UI 명세

### 9.1 반응형 디자인 원칙

본 명세는 **모바일 퍼스트 (Mobile-First)** 접근법을 따릅니다. 이는 다음과 같은 원칙을 의미합니다:

1. **기본 스타일은 모바일용**: CSS 의 기본 스타일은 모바일 (320px) 에 최적화되어 작성됩니다.
2. **점진적 향상 (Progressive Enhancement)**: 더 큰 화면으로 갈수록 기능과 레이아웃이 향상됩니다.
3. **미디어 쿼리는 min-width 사용**: `@media (min-width: 768px)` 형태로 브레이크포인트를 정의합니다.

**핵심 원칙:**

| 원칙 | 설명 |
|------|------|
| **유연한 레이아웃** | 고정 픽셀 대신 상대 단위 (%, rem, vw/vh) 사용 |
| **유연한 이미지** | `max-width: 100%`로 컨테이너에 맞게 조정 |
| **미디어 쿼리** | 콘텐츠 기반 브레이크포인트 사용 |
| **모바일 퍼스트** | 모바일 환경 우선 설계, 점진적 향상 |
| **터치 퍼스트** | 터치 타겟 최소 44×44px 보장 |
| **성능 최적화** | 모바일 네트워크 고려, 지연 로딩 적용 |

| 핀치 | 줌 (비활성화 권장) | 전체 화면 |
| 스와이프 좌우 | 네비게이션/액션 | TodoCard, 이미지 갤러리 |
| 스와이프 상하 | 스크롤 | 전체 화면, 모달 |
| 롱 프레스 | 컨텍스트 메뉴/선택 | TodoCard |
| 풀투리프레시 | 새로고침 | TodoList |

**스와이프 액션 구현 가이드:**

```javascript
// TodoCard 스와이프 예시
const handleTouchStart = (e) => {
  touchStartX = e.touches[0].clientX;
};

const handleTouchMove = (e) => {
  const touchEndX = e.touches[0].clientX;
  const diff = touchStartX - touchEndX;
  
  if (diff > 100) {
    // 왼쪽 스와이프: 삭제 액션 표시
    showDeleteAction();
  } else if (diff < -100) {
    // 오른쪽 스와이프: 완료 액션 표시
    showCompleteAction();
  }
};
```

- **GPU 가속**: opacity, transform 은 GPU 레이어에서 처리

```css
/* 모바일에서 애니메이션 최적화 */
.fab {

    animation: none !important;
  }
}
```
